  ERROR 1.1 — Prisma v7 Breaking Change (url not supported in schema)
────────────────────────────────────────────────────────────────────────────────

  ERROR MESSAGE:
  ─────────────────────────────────────────────────────────────────────────
  Error: Prisma schema validation - (get-config wasm)
  Error code: P1012
  error: The datasource property `url` is no longer supported in schema files.
  Move connection URLs for Migrate to `prisma.config.ts`
  ─────────────────────────────────────────────────────────────────────────

  CAUSE:
  Prisma v7 removed support for `url` in schema.prisma datasource block.
  The global prisma CLI on Render/CI was v7 but our code used v6 syntax.

  SOLUTION:
  Pin Prisma to v6 in package.json:

    npm install prisma@6.19.3 @prisma/client@6.19.3

  Then verify package.json shows:
    "prisma": "^6.19.3"
    "@prisma/client": "^6.19.3"

  PREVENTION:
  Always pin exact Prisma versions. Never use "latest" for Prisma.
  Add to package.json:
    "prisma": "6.19.3"        ← exact, no ^ or ~
    "@prisma/client": "6.19.3"

  ERROR 1.2 — Prisma Cannot Reach Database (P1001)
────────────────────────────────────────────────────────────────────────────────

  ERROR MESSAGE:
  ─────────────────────────────────────────────────────────────────────────
  Error: P1001: Can't reach database server at
  `db.pahfxurtlprpjdqnquav.supabase.co:5432`
  ─────────────────────────────────────────────────────────────────────────

  CAUSE:
  Supabase blocks direct connections on port 5432 from outside networks
  (Termux/Kali on mobile data). The direct URL only works from trusted IPs.

  SOLUTION:
  Run migrations via Supabase SQL Editor instead:

    STEP 1: Generate migration SQL locally:
    npx prisma migrate diff \
      --from-empty \
      --to-schema-datamodel prisma/schema.prisma \
      --script > migration.sql

    STEP 2: Go to Supabase SQL Editor:
    https://supabase.com/dashboard/project/YOUR_PROJECT_REF/sql/new

    STEP 3: Run PostGIS first:
    CREATE EXTENSION IF NOT EXISTS postgis;

    STEP 4: Copy migration.sql content → paste → Run

  ALTERNATIVE:
  Use Supabase connection pooler URL for migrations:
    DATABASE_URL=postgresql://postgres.REF:PASS@aws-0-eu-west-1.pooler.supabase.com:6543/postgres

  ERROR 1.5 — Prepared Statement Already Exists (P42P05)
────────────────────────────────────────────────────────────────────────────────

  ERROR MESSAGE:
  ─────────────────────────────────────────────────────────────────────────
  Raw query failed. Code: `42P05`.
  Message: `ERROR: prepared statement "s0" already exists`
  ─────────────────────────────────────────────────────────────────────────

  CAUSE:
  Using $queryRaw with connection pooler (PgBouncer in transaction mode).
  PgBouncer does not support prepared statements.

  SOLUTION:
  Replace $queryRaw with $executeRaw or use model queries instead:

  WRONG:
    await prisma.$queryRaw`SELECT 1`

  CORRECT:
    await prisma.$executeRaw`SELECT 1`
    // OR use model query:
    await prisma.superAdmin.count()

  For health checks use try/catch with fallback:
    try {
      await prisma.$executeRaw`SELECT 1`
    } catch {
      await prisma.superAdmin.count()
    }

  ERROR 2.1 — PathError Missing Parameter Name (Express 5 Wildcard)
────────────────────────────────────────────────────────────────────────────────

  ERROR MESSAGE:
  ─────────────────────────────────────────────────────────────────────────
  PathError [TypeError]: Missing parameter name at index 1: *;
  visit https://git.new/pathToRegexpError for info
      at consumeUntil (.../path-to-regexp/dist/index.js:108:27)
  ─────────────────────────────────────────────────────────────────────────

  CAUSE:
  Express 5 no longer supports wildcard `*` in route paths.
  Old Express 4 syntax: app.use('*', handler)
  Express 5 removed this.

  SOLUTION:
  Replace wildcard route with a plain middleware (no path):

  WRONG (Express 4):
    app.use('*', (req, res) => {
      res.status(404).json({ error: 'Not found' })
    })

  CORRECT (Express 5):
    app.use((req, res) => {
      res.status(404).json({
        error: 'Route not found',
        path: req.originalUrl,
        method: req.method,
      })
    })

  ERROR 4.1 — Authentication Failed
────────────────────────────────────────────────────────────────────────────────

  ERROR MESSAGE:
  ─────────────────────────────────────────────────────────────────────────
  remote: No anonymous write access.
  fatal: Authentication failed for
  'https://github.com/Kihega/Final_Dissertation/'
  ─────────────────────────────────────────────────────────────────────────

  CAUSE:
  GitHub no longer accepts account passwords for git operations.
  Must use Personal Access Token (PAT).

  SOLUTION:
  STEP 1: Create PAT on GitHub:
    → github.com → Profile → Settings → Developer settings
    → Personal access tokens → Tokens (classic)
    → Generate new token → select: repo, workflow
    → Copy token (shown only once)

  STEP 2: Configure git to use token:
    git config --global credential.helper store
    git push origin dev
    # Enter: username = Kihega
    # Enter: password = paste your PAT token

  Token is saved after first use. No need to re-enter.

  STEP 3: Update remote URL format:
    git remote set-url origin https://Kihega@github.com/Kihega/Final_Dissertation.git

────────────────────────────────────────────────────────────────────────────────
  ERROR 5.1 — Redis Connection Closed
────────────────────────────────────────────────────────────────────────────────

  ERROR MESSAGE:
  ─────────────────────────────────────────────────────────────────────────
  ❌ Redis retry limit reached
  ⚠️ Redis connection closed
  "redis": "error: Connection is closed."
  ─────────────────────────────────────────────────────────────────────────

  CAUSE 1: Wrong Redis URL protocol.
  Upstash requires SSL (rediss://) not plain (redis://).

  SOLUTION 1: Fix URL in .env and Render environment:
    WRONG:   redis://default:PASSWORD@host:6379
    CORRECT: rediss://default:PASSWORD@host:6379
             (note double 's' in rediss)

  CAUSE 2: Redis client being initialized multiple times.

  SOLUTION 2: Add singleton check in redis.js:
    let redis = null
    function connectRedis() {
      if (redis && redis.status === 'ready') return redis
      redis = new Redis(process.env.REDIS_URL, { ... })
      return redis
    }

  CAUSE 3: lazyConnect not configured properly.

  SOLUTION 3: Use enableReadyCheck and proper retry strategy:
    redis = new Redis(process.env.REDIS_URL, {
      maxRetriesPerRequest: 3,
      enableReadyCheck: true,
      retryStrategy(times) {
        if (times > 3) return null
        return Math.min(times * 500, 2000)
      },
    })
