  NBS-018 | MOBILE DEVICE BINDING + GPS GEO-FENCE SETUP
════════════════════════════════════════════════════════════════════════════════

  Owner  : Member 2 (Mobile — React Native/Expo)
  Points : 8  (largest story of the sprint — 2 members pair where possible)

  USER STORY:
  As a new Village Officer, when I enter my one-time token in the mobile app,
  the app silently binds to my device and records my GPS location to establish
  my village geo-fence, so I am permanently tied to one device and one location.

  ACCEPTANCE CRITERIA:
  Given I open the officer mobile app for the first time and enter my token
  When the token is validated by the backend
  Then the app shows a progress screen ("Setting up your device...")
  And in the background:
    a) Device fingerprint is computed: HMAC-SHA256(deviceId + OS + hardwareId)
    b) GPS coordinates are captured (expo-location)
    c) Both are sent to the backend
    d) Backend verifies GPS is within the pre-assigned village boundary
    e) Backend stores device fingerprint + marks officer as device-bound
  And when setup is complete, the app navigates to the profile setup page
  And if GPS is outside the village boundary, setup fails with an error message
  And if the device is already bound to another officer, setup is rejected

  TASKS — BACKEND:
  [ ] Create POST /api/officers/village/bind-device
      { token, deviceFingerprint, gpsLat, gpsLng }
  [ ] Verify token (hash match + not expired + not yet bound)
  [ ] Compute: ST_Within(ST_Point(gpsLng, gpsLat), village geo_fence_polygon)
      → if false, return 400 { error: "GPS outside assigned village boundary" }
  [ ] If unique device fingerprint and inside boundary:
      → update village_officer: device_fingerprint, device_bound_at
      → invalidate token (single use)
      → return 200 { message: "Device bound successfully" }
  [ ] Write Jest test with mock PostGIS geometry queries

  TASKS — MOBILE (React Native + Expo):
  [ ] Create /welcome screen with token text input
  [ ] On token submit → show "Setting up your device..." loading screen
  [ ] Import expo-location → request foreground location permission
      → if denied → show error "Location permission required" + exit
  [ ] Import expo-device → get device type, OS, and deviceId
  [ ] Compute fingerprint: sha256(deviceId + OS + modelName) using crypto-js
  [ ] Call POST /api/officers/village/bind-device with fingerprint + GPS
  [ ] Handle responses:
      - 200: navigate to /profile-setup
      - 400 GPS outside boundary: "You are outside your assigned village area"
      - 400 already bound: "This device is already registered to another officer"
      - 401 invalid token: "Invalid or expired activation token"
  [ ] Store deviceFingerprint in expo-secure-store for future session checks
  [ ] Create WatermelonDB local schema for offline data sync (setup only)

  NBS-019 | GEO-FENCE ENFORCEMENT ON EVERY SESSION
════════════════════════════════════════════════════════════════════════════════

  Owner  : Member 2 (Mobile) + Member 1 (Backend middleware)
  Points : 5

  USER STORY:
  As the system, every time an officer opens the app or starts a new session,
  I verify their GPS location and device fingerprint to ensure they are inside
  their assigned boundary and on their bound device.

  ACCEPTANCE CRITERIA:
  Given I am a bound Village Officer opening the app
  When I attempt to log in
  Then the app captures my current GPS and device fingerprint
  And sends them to the backend with my login credentials
  And if GPS is inside my village boundary AND device matches: login succeeds
  And if GPS is outside boundary: "You are not in your assigned village area"
  And if device fingerprint doesn't match: "Unauthorized device detected"
  And the geo-fence polygon is cached in expo-secure-store for offline checks

  TASKS — BACKEND:
  [ ] Modify POST /api/auth/login (for officer role) to accept:
      { email, password, deviceFingerprint, gpsLat, gpsLng }
  [ ] After password validation → check deviceFingerprint matches stored hash
  [ ] Run ST_Within check against stored geo_fence_polygon
  [ ] If both pass → issue JWT (role=village_officer, scope=village_id)
  [ ] Return geo_fence_polygon in login response (for offline caching)
  [ ] Log GPS coordinates in audit_logs for every officer login

  TASKS — MOBILE:
  [ ] On app open → retrieve stored deviceFingerprint from expo-secure-store
  [ ] Capture current GPS via expo-location
  [ ] Include fingerprint + GPS in all login API calls
  [ ] On login success → cache geo_fence_polygon GeoJSON in expo-secure-store
  [ ] For offline mode: check GPS against cached polygon using
      @turf/boolean-point-in-polygon (client-side geo-fence check)
  [ ] Show persistent location status bar in app: "Inside village ✓" or "⚠ Outside area"
