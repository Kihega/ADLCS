  USER STORY:

  ACCEPTANCE CRITERIA:
  LEVEL 1 — NATIONAL: (done in NBS-050)
  LEVEL 2 — REGION:

      I see: region name, total population, gender ratio, pyramid, trend,

      "→ View by District" button

  LEVEL 3 — DISTRICT:
      I select a district from a list or the density chart
      I see: district name, total pop, gender ratio, pyramid,
             ward density breakdown (%), buildings, infra
      "→ View by Ward" button

  LEVEL 4 — WARD:
      I select a ward
      I see: ward name, total pop, gender ratio, pyramid,
             village density breakdown (%), buildings, infra
      "→ View by Village" button

  LEVEL 5 — VILLAGE:
      I select a village
      I see: village name, total pop, gender ratio, pyramid,

      buildings and infrastructure count

  NOTE: No PII at any level. All data from Redis cache only.

════════════════════════════════════════════════════════════════════════════════

  Owner  : Member 1 (Backend)
  Points : 2

  ACCEPTANCE CRITERIA:
  Given any client makes API requests
  When the rate limit is exceeded
  Then the server returns 429 Too Many Requests with Retry-After header
  And global limit: 100 requests per 15 minutes per IP
  And auth routes (/api/auth/*): 10 requests per 15 minutes per IP
  And admin creation routes: 20 requests per hour per IP
  And rate limit state is stored in Redis (not memory, works across restarts)
