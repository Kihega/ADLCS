  NBS-011 | FIRST SUPER ADMIN BOOTSTRAP
════════════════════════════════════════════════════════════════════════════════

  Owner  : Member 1 (Backend)
  Points : 3

  USER STORY:

  project initialization so that the system is not orphaned with no admin.

  ACCEPTANCE CRITERIA:

  Then a super admin account is created using the provided employee_id, NIDA,

  TASKS — BACKEND:

  [ ] Middleware: check if any super_admin row exists → if yes, return 403
  [ ] Call NIDA mock service (real if available) with NIDA number
  [ ] Hash password with bcrypt (rounds=12)
  [ ] Insert super_admin record (status=active, created_by=null)
  [ ] Return success message (do NOT return password or token in response)
  [ ] Write Jest test: bootstrap creates admin / returns 403 on second call

  NBS-021 | LOGIN ENDPOINT + JWT + MFA
════════════════════════════════════════════════════════════════════════════════

  Owner  : Member 1 (Backend) + Member 2 (Frontend)
  Points : 5

  USER STORY:
  As any admin or public user, I can log in with my email and password and
  optionally enable MFA for additional security.

  ACCEPTANCE CRITERIA:
  Given I submit valid email and password
  When login is processed
  Then I receive a short-lived access token (15 min) and a refresh token (7 days)
  And the refresh token is set as an HttpOnly cookie
  And if MFA is enabled, I am asked for a TOTP code or SMS OTP before access
  And after 5 failed login attempts my account is locked with exponential backoff

  TASKS — BACKEND:
  [ ] Create POST /api/auth/login { email, password, role }
  [ ] Validate email + bcrypt.compare password hash
  [ ] If mfa_enabled=true → generate SMS OTP (Africa's Talking) or
      expect TOTP code from authenticator app
  [ ] If MFA passed → sign JWT access token (RS256, 15m TTL)
  [ ] Set refresh token as HttpOnly Secure SameSite=Strict cookie (7d TTL)
  [ ] Track failed attempts in Redis (key: login_fails:email)
      After 5 fails: lock for 1min, 5min, 15min, 1hr, then require admin reset
  [ ] Create POST /api/auth/mfa/verify { otp, tempToken }
  [ ] Create POST /api/auth/mfa/setup → returns TOTP QR code URL
  [ ] Write Jest tests: valid login / wrong password / MFA / lockout

  TASKS — FRONTEND (Super Admin + District Admin web):
  [ ] Create /login page with email + password form
  [ ] If server returns mfa_required: show OTP input step
  [ ] Store access token in memory (NOT localStorage)
  [ ] Auto-refresh access token 1 min before expiry using React Query


════════════════════════════════════════════════════════════════════════════════
  NBS-022 | TOKEN REFRESH + SESSION EXPIRY MIDDLEWARE
════════════════════════════════════════════════════════════════════════════════

  Owner  : Member 1 (Backend)
  Points : 3

  ACCEPTANCE CRITERIA:
  Given my access token has expired
  When my app sends a request
  Then the backend middleware returns 401 Unauthorized
  And my frontend silently calls POST /api/auth/refresh
  And receives a new access token using the refresh token cookie
  And retries the original request transparently
  And if the refresh token is also expired, I am redirected to login

  TASKS:
  [ ] Create authenticateJWT middleware:
      → verify access token → attach user to req.user → call next()
      → on expiry → return 401 { code: "TOKEN_EXPIRED" }
  [ ] Create POST /api/auth/refresh:
      → read refresh token from HttpOnly cookie
      → verify → issue new access token
      → rotate refresh token (new cookie)
  [ ] Create POST /api/auth/logout:
      → clear refresh token cookie
      → invalidate refresh token in Redis
  [ ] Add axios interceptor in web frontend:
      on 401 TOKEN_EXPIRED → call /auth/refresh → retry original request
      on refresh failure → redirect to /login
