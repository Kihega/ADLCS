--------------------------------------------------------------------------------
1. PROJECT OVERVIEW
--------------------------------------------------------------------------------

Project Name      : Automated Digital Live Census System (ADLCS)
Client / Owner    : Government of Tanzania / Academic Dissertation
Methodology       : Agile Scrum
Duration          : 16 Weeks
Platforms         : Mobile (Android & iOS) + Web Dashboard

Technology Stack:
  - Frontend      : Flutter (Dart) — Mobile & Web
  - Backend       : Node.js with Express.js
  - Database      : PostgreSQL
  - Cache / Sync  : Redis
  - API           : REST API + WebSockets
  - Security      : RBAC + SSL/TLS
  - Mapping       : GIS Integration
  - Offline Mode  : SQLite for data collection in rural areas
  - Version Control: Git


--------------------------------------------------------------------------------
2. SCRUM ROLES
--------------------------------------------------------------------------------

  Role                  Responsibility
  ---------------       -------------------------------------------------------
  Product Owner         Researcher / Dissertation Supervisor.
                        Defines priorities and accepts deliverables.

  Scrum Master          Developer (Student Researcher).
                        Facilitates sprints and removes blockers.

  Development Team      Developer(s) responsible for Flutter app,
                        Node.js backend, PostgreSQL database, Redis, and WebSockets.

  Stakeholders          RITA, NIDA, NBS, Immigration Department,
                        Local Government Authorities, Citizens.

4. SPRINT PLAN
--------------------------------------------------------------------------------

Sprint Duration   : 2 Weeks per Sprint
Total Sprints     : 9 (Sprint 0 to Sprint 8)
Total Duration    : 16 Weeks


====================================================
SPRINT 0 — PROJECT SETUP & REQUIREMENTS
Weeks: 1 – 2 | Story Points: 15
====================================================

Sprint Goal:
  Establish project foundation, gather all requirements, and
  set up development environments.

Tasks:
  [ ] Conduct stakeholder interviews and distribute questionnaires
  [ ] Create Product Backlog with prioritized user stories
  [ ] Set up Git repository and branching strategy
  [ ] Initialize Flutter project (mobile + web)
  [ ] Set up Node.js + Express.js backend
  [ ] Configure PostgreSQL and Redis locally
  [ ] Design initial Use Case Diagram

User Stories Covered: US-01

Deliverables:
  - Requirements Document
  - Completed Product Backlog
  - Development Environment Ready
  - Git Repository Initialized


====================================================
SPRINT 1 — SYSTEM DESIGN
Weeks: 3 – 4 | Story Points: 26
====================================================

Sprint Goal:
  Complete system architecture, database design, and wireframes.

Tasks:
  [ ] Design overall system architecture diagram
  [ ] Design ER Diagram and PostgreSQL schema
  [ ] Document REST API endpoints
  [ ] Define WebSocket event contracts
  [ ] Design UML Class & Sequence Diagrams
  [ ] Create Flutter mobile app wireframes
  [ ] Create web dashboard wireframes

User Stories Covered: US-02, US-03, US-04

Deliverables:
  - System Architecture Diagram
  - ER Diagram and Database Schema
  - REST API Documentation

  - Mobile & Web Wireframes

6. DEFINITION OF DONE (DoD)
--------------------------------------------------------------------------------

A Sprint item is considered DONE when ALL of the following are true:

  [1] Code is written and follows project coding standards
  [2] Unit tests are written and pass successfully
  [3] Feature is integrated and works end-to-end
  [4] Code is reviewed and merged to main branch
  [5] Feature tested on mobile and web platforms
  [6] Relevant section is documented in dissertation

7. KEY RISKS & MITIGATION STRATEGIES
--------------------------------------------------------------------------------

  Risk                              Impact  Mitigation Strategy
  --------------------------------  ------  ----------------------------------
  Poor internet connectivity in     High    Implement offline-first Flutter app
  rural areas                               with local SQLite sync queue

  Limited interoperability with     High    Use standard REST API with JSON
  RITA, NIDA, and NBS systems               data format for integration

  Data privacy and security         High    Enforce RBAC, SSL/TLS, and
  breaches                                  encrypted database fields

  Scope creep during development    Medium  Maintain strict sprint backlog
                                            control and freeze scope per sprint

  Redis/WebSocket integration       Medium  Build and test prototype in
  complexity                                Sprint 5 before full rollout

  Stakeholder availability for UAT  Medium  Schedule UAT early and
                                            provide test scripts in advance
