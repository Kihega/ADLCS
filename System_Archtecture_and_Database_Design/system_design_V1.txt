  ──────────────────────────────────────────────────────────────────────────
  [node-cron job — every 15 min on Render backend]
        │
        ▼
  Query PostgreSQL → aggregate population by all geo levels
        │
        ▼
  Write results to Upstash Redis (TTL = 15 min)
        │
        ▼
  Citizen app polls GET /api/analytics/national (or region/etc.)
        │
        ├─► Cache HIT  → return Redis data instantly
        └─► Cache MISS → query PostgreSQL → write Redis → return

          │  • Email sent via Resend API with token + dashboard URL
          ▼
  STEP 4: New Admin Opens Link → Enters Token
          │  • Token validated: hash match + not expired + not used
          ▼
  STEP 5: Profile Setup
          │  • Password + confirm, profile photo (Cloudinary upload)
          │  • Mobile number, Department
          ▼
  STEP 6: Token Invalidated → Account Active
          ▼
  STEP 7: Login (email + password) → Optional MFA (TOTP or SMS OTP)
          ▼
  [Super Admin Dashboard — full national access]

          ▼

          ▼

          ▼

          │
          │  [BACKGROUND — runs silently after token accepted]
          │  a) Capture device fingerprint (HMAC of device ID + OS + hardware)
          │  b) Record GPS coordinates

          │  d) Fetch and cache village geo-fence polygon
          │  e) All checks pass → unlock profile setup page
          ▼

          ▼
  STEP 6: Account active — geo-locked
          │  On every login: GPS validated against geo-fence polygon
          │  Outside boundary → session blocked
          │  Wrong device → session blocked
          ▼
  [Village Officer App — village-scoped, device-locked]

────────────────────────────────────────────────────────────────────────────────
  2.5  CITIZEN REGISTRATION FLOW  (Village Officer App)
────────────────────────────────────────────────────────────────────────────────

  STEP 1: Enter National ID → NIDA fetch → check age ≥ 18
  STEP 2: Check NIDA disability flag → confirm or update
  STEP 3: Biometric capture → photo (Expo Camera), signature (pad)
  STEP 4: Enter: education, street, house no., marital status,
          children IDs, occupation(s)
  STEP 5: Optional: land acres, animals, health insurance,
          comm. devices, social funds
  STEP 6: Review & confirm → issue National ID card (PDF)
          Citizen status = ALIVE
          ID card expires: issued_date + 10 years


────────────────────────────────────────────────────────────────────────────────
  2.6  BIRTH REGISTRATION FLOW  (Hospital Officer App)
────────────────────────────────────────────────────────────────────────────────

  STEP 1: Enter child name (first, middle, surname), gender, DOB
  STEP 2: Father NID + Mother NID → validate in system
  STEP 3: Facility ID auto-attached + officer NID linked
  STEP 4: System generates unique National ID for newborn
          Child status = ALIVE, age tracking starts from DOB
  STEP 5: Birth certificate PDF generated + QR signed
          Pushed to RITA API (rita_synced = true)
  NOTE:   Child must visit Village Officer at age 18+ for full reg.


────────────────────────────────────────────────────────────────────────────────
  2.7  DEATH REGISTRATION FLOW
────────────────────────────────────────────────────────────────────────────────

  OUTSIDE HOSPITAL (Village Officer):
  • Enter citizen NID → fetch info
  • Enter: date of death, cause
  • Status → DECEASED, age frozen
  • Death certificate PDF + QR → pushed to RITA

  INSIDE HOSPITAL (Hospital Officer):
  ├─► INFANT: father NID + mother NID + date + cause
  └─► ADULT:  citizen NID + cause + date + duty officer NID
  • Certificate generated → RITA sync
  • Citizen status → DECEASED


────────────────────────────────────────────────────────────────────────────────
  2.8  MARRIAGE REGISTRATION FLOW  (Village Officer App)
────────────────────────────────────────────────────────────────────────────────

  STEP 1: Enter Husband NID + Wife NID → verify both citizens
  STEP 2: Validate rules:
          Male  → multiple wives allowed simultaneously
          Female → only 1 active husband at a time
                   Exception: husband DECEASED or last marriage UNREGISTERED
  STEP 3: System generates marriage_id + certificate PDF → RITA sync
  STEP 4: Marriage status = ACTIVE

  UNREGISTER FLOW:
  Enter marriage_id → submit dissolution request → authorized
  institution verifies → provides reason → on approval:
  status → DISSOLVED, wife's lock released


────────────────────────────────────────────────────────────────────────────────
  2.9  MIGRATION REGISTRATION FLOW  (Village Officer App)
────────────────────────────────────────────────────────────────────────────────

  STEP 1: Source officer enters citizen NID → returns current residence
  STEP 2: Enter new: Region → District → Ward → Village + reason
  STEP 3: Migration request created (status: PENDING, expiry: +30 days)
          Target village officer notified (push notification)
  STEP 4: Citizen physically visits target village within 30 days
  STEP 5: Target officer confirms in app
          → Citizen residence updated to new village
          → Population counters auto-updated in Redis
          → If no action in 30 days → status = EXPIRED (auto by cron)

  SECTION 4 — SECURITY & PRIVACY ARCHITECTURE
████████████████████████████████████████████████████████████████████████████████

  PASSWORDS        : bcrypt cost factor 12
  JWT ACCESS TOKEN : 15-minute TTL, RS256 signed
  REFRESH TOKEN    : 7 days TTL, rotated on use, stored HttpOnly cookie
  ONE-TIME TOKENS  : SHA-256 hash stored, 24h TTL, single-use only
  MFA              : TOTP (RFC 6238) optional / SMS OTP 6-digit 5min TTL
  ACCOUNT LOCKOUT  : 5 failed attempts → exponential backoff → admin reset

  DEVICE BINDING   : HMAC-SHA256(device_id + OS + hardware_hash)
                     Bound at first token activation, cannot change
                     without District Admin reset

  GEO-FENCE        : PostGIS ST_Within() on every session start
                     Polygon cached in Upstash Redis (encrypted)
                     Mock GPS detection → session blocked

  ENCRYPTION       : TLS 1.3 in transit
                     AES-256-GCM at rest (fingerprints, NIDA numbers)
                     Cloudinary encrypted storage (photos, PDFs)

  RBAC             : Row-Level enforced by Prisma middleware
                     Public app: zero PII, aggregated stats only

  AUDIT            : Append-only audit_logs table
                     Every write operation logged with actor + GPS + IP

  NETWORK          : Rate limiting (express-rate-limit + Redis)
                     CORS restricted to known origins
                     Helmet.js security headers on Express
